## 🔴 Critical (breaks functionality)

- `trv_story.js:136` + `trv_story.js:148` — Function-name collision / unintended recursion.
  The local `function getCountryName()` (no args) shadows the global
  `getCountryName(countryId)` from `bcd_services.js:196` throughout this file. For
  multi-country stories (`countryId.length > 1`), line 148 calls `getCountryName(country)`
  meaning to look up a country's display name, but it re-invokes the LOCAL function, which
  ignores the argument and returns the entire `<ul>…</ul>`/`<div>` block. Result: the
  "back to country" list on multi-country story pages renders broken nested markup instead
  of country names. Single-country stories (the `else` branch, line 153) are unaffected
  because they never call it. Rename the local function (e.g. `storyBackLinks`).

- `bcd-content.js:8` — Main database load has no error handler.
  `$.getJSON("DATA/globaldb.json", function(result){ processMyJson(result); callback();})`
  has no `.fail()`. This runs on every navigation to a visited page (country/city/story via
  `populateContent`). If the 1.9 MB file fails to load or is malformed, the page silently
  hangs with no feedback. Add a `.fail()` with a user-visible error.

- `bcd_onload.js:38` — Home-page load has no error handler.
  `window.onload = function() {$.getJSON("DATA/onload.json", processMyJson)}` — if
  `onload.json` fails, the entire site renders nothing and no error is shown.
